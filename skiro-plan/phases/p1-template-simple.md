# skiro-plan | always load | ~600 tok

## 세션 계획 템플릿

### 목표 (Goals)
- 명확하고 측정 가능한 형태로 기술
- "잘 되면 좋겠다" 금지 → "X를 실행하면 Y 결과가 나온다"
- 예: "AK60-6 1번 모터 토크 5Nm 인가 → 각도 응답 측정"

### 전제 조건 (Preconditions)
- 필요한 하드웨어 상태
- 필요한 소프트웨어 빌드 상태
- 필요한 파일/데이터

### 단계 (Steps)
1단계부터 순서대로. 각 단계는 독립적으로 검증 가능해야 한다.
단계당 예상 시간 명시.

### 검증 기준 (Acceptance Criteria)
각 목표에 대한 합격/불합격 기준.
"동작함" 금지 → "콘솔에 X 출력 + Y 측정값 ±Z 이내"

### 롤백 계획 (Rollback)
단계별 실패 시 어떻게 되돌아오는가.
하드웨어 작업이면 전원 차단 순서 명시.

### 예상 소요 시간
- 준비: N분
- 실행: N분
- 검증: N분
- 롤백 여유: N분

## 빠른 판단 체크리스트

```
계획 수립 전:
  □ skiro-safety 분석이 필요한 하드웨어 작업인가?
  □ current-experiment.json이 이미 있으면 덮어쓰지 않는다
  □ 이전 세션 learnings에서 관련 교훈 확인

계획 수립 후:
  □ 목표가 SMART 기준을 충족하는가?
  □ 롤백 계획이 명시되었는가?
  □ safety_required 플래그가 올바른가?
```
