# p2-hw-risk.md — 하드웨어 위험 분석
# skiro-plan | hardware/milestone tier | ~540 tok

## 하드웨어 위험 체크리스트

### 전력 계통
```
□ 전원 공급 순서: 컨트롤러 먼저, 액추에이터 나중
□ 최대 전류 소비 계산: Σ(모터 스톨 전류) < 전원 정격의 80%
□ 비상 전원 차단 경로 확인 (물리 E-stop 위치)
□ 배터리 작업 시: 잔량 > 30%, 셀 밸런스 확인
```

### 기계 계통
```
□ 관절 가동 범위 물리적 스토퍼 확인
□ 케이블 루팅: 가동부에 걸리지 않음
□ 하중 방향: 중력 보상 필요 여부
□ 비상 시 인원 퇴피 경로 확인
```

### 통신 계통 (CAN)
```
□ 버스 종단저항: 양 끝 120Ω 확인
□ CAN ID 충돌 없음 (모터별 ID 고유)
□ 타임아웃 설정 확인 (기본: 250ms)
□ 오실로스코프로 버스 신호 확인 (선택)
```

### H-Walker 전용
```
□ AK60-6 모터 온도 < 60°C (시작 전)
□ Teensy 4.1 펌웨어 버전 확인
□ ZED X Mini 카메라 초기화 완료
□ 로드셀 영점 보정 완료
□ IMU 캘리브레이션 상태 확인
```

## 위험 등급 분류

| 등급 | 조건 | 대응 |
|------|------|------|
| LOW | 전원 없는 소프트웨어 테스트 | 계획 그대로 진행 |
| MED | 저전압(< 24V) 단일 모터 | skiro-safety 간이 실행 |
| HIGH | 고전압 or 다축 or 하중 있음 | skiro-safety 전체 실행 필수 |
| CRITICAL | 사람 접촉 가능성 있음 | 별도 안전 프로토콜 수립 |
